# File Organizer CLI

A powerful Node.js command-line tool to analyze your directory, find duplicates, organize files by categories, and cleanup old data.

## Features

- **Scan**: Recursively scan a directory and get detailed statistics (count, size, types, age distribution, largest/oldest files).
- **Duplicates**: Find files with identical content using SHA-256 hashes (efficient streaming for large files).
- **Organize**: Automatically sort files into categories (Documents, Images, Archives, Code, Videos, Other).
- **Cleanup**: Identify and remove files older than a specific number of days with Dry Run support.
- **EventEmitter Architecture**: Real-time progress tracking with progress bars in the console.
- **Performance**: Uses streams for large files (≥10 MB) to minimize memory consumption.

## Installation

1. Ensure you have **Node.js (v20.10.0 or later)** installed.
2. Clone or download this project.
3. Navigate to the project directory.

```bash
cd file-organizer
```

## Usage

You can use `npm run` which passes arguments to `node file-organizer.js`.

### 📊 Scan Directory
Analyze the state of a directory and see detailed group statistics:
```bash
npm run scan -- /path/to/directory
```

### 🔍 Find Duplicates
Identify files with identical content even if names are different:
```bash
npm run duplicates -- /path/to/directory
```

### 📦 Organize Files
Sort files into categorized folders in a target location. Original files are preserved.
```bash
npm run organize -- /source/directory --output /target/directory
```

### 🧹 Cleanup Old Files
Preview files older than a specific date:
```bash
npm run cleanup -- /path/to/directory --older-than 90
```

Actually delete the files using the `--confirm` flag:
```bash
npm run cleanup -- /path/to/directory --older-than 90 --confirm
```

## Technical Implementation

- **Asynchronous Operations**: All file operations utilize `fs/promises` or streaming for non-blocking execution.
- **Stream Processing**: Files ≥10MB are handled via `fs.createReadStream` and `pipeline` for memory efficiency.
- **Duplicate Detection**: Uses `crypto.createHash('sha256')` with streaming data chunks.
- **Event-Driven**: Decoupled architecture where logic classes emit events (`file-found`, `copy-start`, etc.) and the CLI entry point handles console output.
- **Error Handling**: Specific handling for common filesystem errors like `ENOENT` (not found) and `EACCES` (permission denied).
- **Conflict Resolution**: During organization, unique filenames are generated by adding suffixes like `(1)`, `(2)` if a file already exists.
